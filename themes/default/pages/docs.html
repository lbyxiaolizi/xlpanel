Everything you need to know to get started with OpenHost

Introduction

OpenHost is a modern, open-source hosting and billing management system built with Go. It's designed to be a powerful alternative to proprietary solutions like WHMCS and Blesta, offering enterprise-grade features with the flexibility and transparency of open source.

Key Features

  • High Performance - Built with Go for blazing fast response times
  • Secure by Design - Enterprise-grade security features built-in
  • Plugin Architecture - Extend functionality with gRPC plugins
  • Precise Billing - Accurate financial calculations using decimal arithmetic
  • RESTful API - Complete API with Swagger documentation

Installation

OpenHost can be installed in several ways. Choose the method that works best for your environment.

Prerequisites

  • Go 1.21 or higher
  • PostgreSQL 14+ or SQLite 3.35+
  • 512MB RAM minimum (2GB+ recommended)

Using Go Install

go install github.com/openhost/openhost@latest

Using Docker

docker pull openhost/openhost:latest
docker run -d -p 6421:6421 openhost/openhost:latest

Building from Source

git clone https://github.com/openhost/openhost.git
cd openhost
make build
./bin/openhost serve

Quick Start

Get OpenHost up and running in just a few minutes.

1

Run the Installer

Navigate to http://localhost:6421/install to access the web installer.

2

Configure Database

Choose between SQLite (default) or PostgreSQL and enter your connection details.

3

Create Admin Account

Set up your administrator email and password.

4

Start Using OpenHost

Log in to the admin panel and start configuring your products and services.

Configuration

OpenHost uses a configuration file for most settings. The default location is config.yaml in the application directory.

Example Configuration

server:
  host: "0.0.0.0"
  port: 6421
  mode: "production"

database:
  driver: "sqlite"
  path: "./data/openhost.db"

security:
  session_secret: "your-secret-key"
  bcrypt_cost: 12

mail:
  driver: "smtp"
  host: "smtp.example.com"
  port: 587
  username: "noreply@example.com"
  password: "your-password"

Environment Variables

All configuration options can also be set via environment variables with the OPENHOST_ prefix:

export OPENHOST_SERVER_PORT=8080
export OPENHOST_DATABASE_DRIVER=postgres
export OPENHOST_DATABASE_URL="postgres://user:pass@localhost/openhost"
